`types.ts`:
Define and export:
```typescript
import { z } from 'zod'

export const IGProfileSchema = z.object({
  bio: z.string().nullable(),
  follower_count: z.number().nullable(),
  following_count: z.number().nullable(),
  post_count: z.number().nullable(),
  is_private: z.boolean(),
  profile_pic_url: z.string().nullable(),
  recent_captions: z.array(z.string()),
})

export type IGProfile = z.infer<typeof IGProfileSchema>

export interface IGScrapeResult {
  success: boolean
  profile: IGProfile | null
  error?: string
}
```

`scraper.ts`:
Create `scrapeIGProfile(handle: string): Promise<IGScrapeResult>`.

The scraper should use a server-side fetch to call an internal scrape endpoint OR directly call the Browserbase Stagehand SDK if available. Since this runs server-side in a Next.js API route:

1. Use the Stagehand SDK directly (import from `@browserbasehq/stagehand` if installed, otherwise use `fetch` to Browserbase REST API).
2. Steps:
   a. Create a Browserbase session
   b. Navigate to `https://www.instagram.com/${handle}/`
   c. Wait for page load (2-3 seconds)
   d. Check for private account / login wall using page content inspection
   e. If public: extract bio, follower/following/post counts, profile pic URL, and up to 12 recent post captions
   f. Close session
3. Return structured IGProfile or error.

IMPORTANT: Check if `@browserbasehq/stagehand` is in package.json. If not, install it. Also need `BROWSERBASE_API_KEY` and `BROWSERBASE_PROJECT_ID` env vars.

If Stagehand SDK is NOT available or too complex to set up server-side, implement a SIMPLER approach:
- Use `fetch` to call Instagram's public JSON endpoint: `https://www.instagram.com/${handle}/?__a=1&__d=dis` (may be blocked)
- OR use the Browserbase REST API directly to create a session, navigate, and extract
- OR implement as a thin wrapper that the MCP tools call during execution (the enrichment happens when an agent runs, not in production)

For the MVP: implement the scraper as an async function that uses Browserbase REST API:
```
POST https://www.browserbase.com/v1/sessions (create session)
POST https://www.browserbase.com/v1/sessions/{id}/navigate (go to IG)
POST https://www.browserbase.com/v1/sessions/{id}/extract (get data)
DELETE https://www.browserbase.com/v1/sessions/{id} (close)
```

Handle errors gracefully: network timeout, login wall detected, private account. Always close the session in a finally block.

For rate limiting: add a check -- if ig_scraped_at is within 7 days for this handle, skip the scrape and return cached data.

`extract-interests.ts`:
Create `extractInterests(igData: IGProfile): Promise<IGInterests>` using Claude (same pattern as `generate-replies.ts`):
- Import Anthropic SDK
- Send bio + recent_captions to Claude claude-sonnet-4-6 with max_tokens: 512
- System prompt: "You are analyzing an Instagram profile for dating intelligence. Extract specific interests, not generic observations."
- User prompt: provide bio + captions, request JSON matching IGInterests shape (hobbies, travel, food, fitness, music, aesthetic, red_flags, conversation_hooks)
- Parse response as JSON, validate with a runtime check
- If fewer than 3 captions, return a minimal result with just bio-derived data
- Handle Claude API errors gracefully

`enrich/route.ts`:
Create POST handler at `/api/matches/[id]/enrich`:
1. Auth check: `supabase.auth.getUser()`, return 401 if not authenticated
2. Fetch the match_profiles row by id, verify user_id matches
3. Check ig_handle exists, return 400 if missing
4. Check ig_scraped_at -- if within 7 days, skip scrape (return cached)
5. Update enrichment_status to 'enriching'
6. Call `scrapeIGProfile(ig_handle)`
7. If scrape succeeds:
   a. Update match_profiles with all IG fields (bio, followers, captions, etc.)
   b. Call `extractInterests(profile)` if enough data (bio + 3+ captions)
   c. Update ig_interests with extraction result
   d. Set enrichment_status = 'done', ig_scraped_at = now()
8. If scrape fails:
   a. Set enrichment_status = 'failed', enrichment_error = error message
9. Return { status: enrichment_status, profile: updated_match }

Add `export const maxDuration = 60` for Vercel Pro timeout (IG scrape takes 8-15s).

Use try/catch/finally pattern -- always update enrichment_status even on error.
