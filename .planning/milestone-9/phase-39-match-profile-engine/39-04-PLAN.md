Implementation:
1. Data sufficiency check: return null if NONE of these are present:
   - igBio (non-empty string)
   - igCaptions with 3+ items
   - conversationHistory with 200+ chars
   - notes (non-empty string)
   If only name + birthday, return null with no API call (avoids generic output per Pitfall 4 from research).

2. Build context string from available data:
   - "Name: {name}"
   - If zodiac: "Zodiac: {sign} - {commsHints}"
   - If IG bio: "Instagram Bio: {igBio}"
   - If IG captions: "Recent Posts: {first 8 captions joined by newline}"
   - If IG interests: "Extracted Interests: {JSON}"
   - If conversation: "Conversation History (last 20 messages): {history}"
   - If notes: "Manual Notes: {notes}"

3. Call Claude claude-sonnet-4-6:
   - System prompt: "You are a dating communication analyst helping a man (Julian, Leo, direct and playful) craft his approach for each match. Analyze the provided data about a woman and return a JSON communication profile. Be SPECIFIC to this person -- never return generic advice like 'she seems friendly'. Every recommendation must tie to something in the data."
   - User prompt: "Analyze this person and return ONLY valid JSON (no markdown, no explanation) matching this exact schema: {CommunicationProfile schema with descriptions of each field}"
   - max_tokens: 512
   - temperature: 0.3 (lower for more consistent structured output)

4. Parse response:
   - Extract text content from Claude response
   - Try JSON.parse
   - Validate against expected shape (disc_estimate is D/I/S/C, best_topics has 3+ items, etc.)
   - If parse fails, return a minimal fallback profile based on zodiac hints only (if available)

5. Return CommunicationProfile or null.

Handle Anthropic API errors (rate limit, timeout) gracefully -- log error, return null. Do NOT throw and crash the enrichment flow.
