  <action>
`/api/matches/route.ts` — POST (create) + GET (list):

POST handler:
1. Auth: `const supabase = await createClient()`, `supabase.auth.getUser()`, 401 if not authed
2. Parse body: { name, source_platform, phone?, birthday?, ig_handle?, notes? }
3. Validate: name required (non-empty), source_platform must be one of tinder/bumble/hinge/irl/other
4. Build insert object with all provided fields
5. If birthday provided:
   a. `getZodiac(birthday)` -> get sign, element, modality, planet
   b. Fetch Julian's birthday from `profiles` table: `supabase.from('profiles').select('date_of_birth').eq('id', user.id).single()`
   c. If Julian's DOB exists: `getZodiac(julianDob)` then `getCompatibility(julianSign, matchSign)`
   d. Add zodiac_sign, zodiac_element, zodiac_modality, zodiac_ruling_planet, zodiac_compatibility_score, zodiac_compatibility_breakdown to insert
6. Set enrichment_status: if ig_handle provided -> 'pending', else -> 'skipped'
7. Insert into match_profiles, return { match, enriching: !!ig_handle }
8. Do NOT use after() for enrichment -- client will call /api/matches/[id]/enrich separately (per research recommendation)

GET handler:
1. Auth check
2. `supabase.from('match_profiles').select('*').eq('user_id', user.id).order('created_at', { ascending: false })`
3. Return array of matches

`/api/matches/[id]/route.ts` — GET (single) + PATCH (update):

GET handler:
1. Auth check
2. Fetch by id, verify user_id matches
3. Return match or 404

PATCH handler:
1. Auth check
2. Fetch by id, verify user_id matches
3. Accept partial updates: any field from MatchProfile (except id, user_id, created_at)
4. If birthday is being updated: recalculate zodiac + compatibility
5. Update and return updated match

Import createClient from '@/lib/supabase/server'. Import getZodiac from '@/lib/zodiac/calculate'. Import getCompatibility from '@/lib/zodiac/compatibility'.
